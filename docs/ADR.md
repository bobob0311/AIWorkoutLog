# Architecture Decision Records

## 철학
운동 기록은 빠르게 입력되어야 하고, 통계는 같은 데이터 기준으로 일관되게 계산되어야 한다. 구현은 TDD를 통해 안정적으로 확장하고, 현재 단계에서는 사용성을 크게 개선하는 방향의 리팩토링을 우선한다.

---

### ADR-001: Next.js App Router + FSD 채택
**결정**: 프론트엔드는 Next.js App Router와 Feature-Sliced Design을 사용한다.  
**이유**: 페이지 라우팅, 서버 연동, 구조적 확장성을 함께 가져가기 좋고 기능 단위 분리가 명확하다.  
**트레이드오프**: 초기에 레이어 구조와 import 규칙이 다소 엄격하게 느껴질 수 있다.

### ADR-002: Emotion 스타일링 채택
**결정**: UI 스타일링은 Emotion을 사용한다.  
**이유**: 컴포넌트와 스타일을 가깝게 유지하면서도 동적 스타일 구성이 쉽다.  
**트레이드오프**: CSS-in-JS 경험이 적으면 초기 진입 비용이 있다.

### ADR-003: Supabase 기반 데이터 저장과 간편 로그인
**결정**: 데이터 저장은 Supabase를 목표로 하고, 현재 MVP 단계는 mock 저장소와 프로필 선택 + 비밀번호 흐름으로 단순화한다.  
**이유**: 백엔드 구조를 빠르게 준비하면서도 사용자 흐름을 먼저 검증할 수 있다.  
**트레이드오프**: 현재 인증 체계와 저장소는 실제 운영 기준으로는 추가 보강이 필요하다.

### ADR-004: 세트 단위 기록과 볼륨 계산 기준 고정
**결정**: 운동 기록은 세트 단위로 저장하고, 볼륨은 `중량 x 횟수` 기준으로 계산한다.  
**이유**: 날짜/주간/월간 통계를 모두 같은 공식으로 계산할 수 있고 분석 정확도가 높다.  
**트레이드오프**: 메모형 기록보다 입력 항목이 많아진다.

### ADR-005: 고정 부위 목록 + 기본 운동 목록 + 개인 운동 추가
**결정**: 부위는 고정 목록으로 관리하고, 운동명은 기본 목록과 사용자 추가 목록을 함께 사용한다.  
**이유**: 통계 일관성과 입력 유연성을 동시에 확보할 수 있다.  
**트레이드오프**: 완전 자유 입력보다 데이터 모델이 조금 더 복잡해진다.

### ADR-006: 날짜 셀 정보는 텍스트 나열 대신 미니 차트 중심으로 표현
**결정**: 달력 날짜 셀의 부위별 정보는 텍스트 나열보다 `총 볼륨 + 부위별 미니 차트` 중심으로 바꾼다.  
**이유**: 총 볼륨과 부위별 집계를 더 빠르게 구분할 수 있고, 작은 셀 안에서도 시각적 우선순위를 만들 수 있다.  
**트레이드오프**: 색상 체계와 차트 밀도가 중요해져 접근성과 가독성 점검이 필요하다.

### ADR-007: 달력은 단일 고정 월이 아니라 월 탐색 가능한 구조로 운영
**결정**: 달력은 이전 월, 다음 월, 이번 달 이동을 지원하는 상태 기반 캘린더로 운영한다.  
**이유**: 기록 앱 핵심 사용성은 여러 월 탐색에 있으며 특정 월 고정은 실사용성에 한계가 크다.  
**트레이드오프**: 월 상태, 모달 상태, 날짜 요약 계산이 함께 움직이므로 상태 관리가 복잡해진다.

### ADR-008: 통계는 preset + 직접 기간 설정을 함께 지원
**결정**: `일 / 주 / 월` preset을 유지하면서 `직접 기간 설정`도 추가한다.  
**이유**: 사용자가 원하는 분석 구간이 고정된 단위로만 끝나지 않기 때문에, 비교 가능한 범위 선택이 필요하다.  
**트레이드오프**: 범위 계산과 fallback 기준을 문서화하지 않으면 구현 간 불일치가 생기기 쉽다.

### ADR-009: 부족 부위는 상대 비교 규칙으로 계산
**결정**: 부족 부위는 `직전 동일 길이 기간 대비 볼륨이 감소했고 현재 기간 볼륨이 낮은 부위` 기준으로 계산한다.  
**이유**: 목표 볼륨 데이터가 없는 상태에서도 사용자가 해석 가능한 약점 정보를 제공할 수 있다.  
**트레이드오프**: 기록이 적은 사용자에게는 결과가 불안정할 수 있어 fallback 규칙이 필요하다.  
**fallback**: 직전 기간 데이터가 부족하면 `현재 기간 기준 볼륨이 낮은 부위`만 보여준다.

### ADR-010: 차트 색상은 부위 단위 팔레트를 공유
**결정**: 부위별 통계, 날짜 셀 미니 차트, 운동 랭킹 차트는 같은 부위 컬러 시스템을 공유한다.  
**이유**: 사용자가 색상만 봐도 부위를 연상할 수 있어 화면 간 의미 연결이 좋아진다.  
**트레이드오프**: 운동별 색이 아니라 부위별 색을 따르므로, 운동 자체를 구분하는 데는 라벨과 정렬이 중요해진다.
