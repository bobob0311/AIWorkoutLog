# UI 사용자 가이드

## 사용자 경험 원칙
1. 달력은 첫 인상에서 운동 기록 앱이라는 목적이 바로 드러나야 한다.
2. 운동 기록 입력은 빠르고 단순해야 한다.
3. 숫자 정보는 장식보다 읽기 쉬워야 한다.
4. 부위별 볼륨은 색상만이 아니라 텍스트와 위치로도 먼저 전달돼야 한다.
5. 모바일에서도 날짜 선택과 기록 입력이 무너지지 않아야 한다.

## 피해야 할 표현
| 금지 사항 | 이유 |
|-----------|------|
| 과한 그라디언트 효과 | 정보 중심 제품의 가독성을 해친다 |
| 지나치게 많은 차트 종류 | MVP 복잡도만 높이고 해석을 방해한다 |
| 달력 칸 안의 과도한 정보 | 중요한 숫자를 읽기 어렵게 만든다 |
| 운동 입력 흐름의 긴 단계 분리 | 기록 속도를 떨어뜨린다 |

## 핵심 화면 가이드
### 로그인 화면
- 프로필 선택을 먼저 보여준다.
- 비밀번호 입력은 한 줄로 단순하게 둔다.
- 새 프로필 추가는 보조 흐름으로 둔다.

### 달력 화면
- 월간 달력이 화면의 중심이다.
- 날짜 칸에는 날짜, 총 볼륨, 부위별 볼륨 2~3개만 표시한다.
- 기록 있는 날짜는 배경과 숫자 강조로 차이를 준다.

### 날짜 기록 모달
- 상단에는 날짜와 총 볼륨을 먼저 보여준다.
- 기존 기록 목록과 입력 폼이 한 흐름 안에서 이어져야 한다.
- 세트 입력은 반복 가능한 구조로 빠르게 추가 가능해야 한다.

### 통계 화면
- 요약 카드를 먼저 보여주고, 비교 정보와 차트를 아래에 둔다.
- 일, 주, 월, 직접 기간 전환은 상단에서 빠르게 바꿀 수 있어야 한다.
- 숫자와 라벨의 의미가 명확해야 한다.
- 많이 한 부위, 부족 부위, 부위별 비교, 운동 랭킹 순서로 해석 흐름을 만든다.

## 컴포넌트
### 카드
```text

background: #ffffff
border: 1px solid #e5e7eb
padding: 20px
```

### 버튼
```text
Primary: 배경이 있는 우선 버튼
Secondary: 연색 배경 + 회색 보더 버튼
Text: 배경 없는 텍스트 버튼
```
