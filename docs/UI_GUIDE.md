# UI 사용자 가이드

## 사용자 경험 원칙
1. 달력은 첫 인상에서 운동 기록 앱이라는 목적이 바로 드러나야 한다.
2. 운동 기록 입력은 빠르고 단순해야 한다.
3. 숫자 정보는 장식보다 읽기 쉬워야 한다.
4. 부위별 볼륨은 색상만이 아니라 텍스트와 위치로도 먼저 전달돼야 한다.
5. 모바일에서도 날짜 선택과 기록 입력이 무너지지 않아야 한다.

## 피해야 할 표현
| 금지 사항 | 이유 |
|-----------|------|
| 과한 그라디언트 효과 | 정보 중심 제품의 가독성을 해친다 |
| 지나치게 많은 차트 종류 | MVP 복잡도만 높이고 해석을 방해한다 |
| 달력 칸 안의 과도한 정보 | 중요한 숫자를 읽기 어렵게 만든다 |
| 운동 입력 흐름의 긴 단계 분리 | 기록 속도를 떨어뜨린다 |

## 핵심 화면 가이드
### 로그인 화면
- 프로필 선택을 먼저 보여준다.
- 비밀번호 입력은 한 줄로 단순하게 둔다.
- 새 프로필 추가는 보조 흐름으로 둔다.

### 달력 화면
- 월간 달력이 화면의 중심이다.
- 날짜 칸에는 날짜, 총 볼륨, 부위별 볼륨 2~3개만 표시한다.
- 기록 있는 날짜는 배경과 숫자 강조로 차이를 준다.

### 날짜 기록 모달
- 상단에는 날짜와 총 볼륨을 먼저 보여준다.
- 기존 기록 목록과 입력 폼이 한 흐름 안에서 이어져야 한다.
- 세트 입력은 반복 가능한 구조로 빠르게 추가 가능해야 한다.

### 통계 화면
- 요약 카드를 먼저 보여주고, 비교 정보와 차트를 아래에 둔다.
- 일, 주, 월, 직접 기간 전환은 상단에서 빠르게 바꿀 수 있어야 한다.
- 숫자와 라벨의 의미가 명확해야 한다.
- 많이 한 부위, 부족 부위, 부위별 비교, 운동 랭킹 순서로 해석 흐름을 만든다.

## 훈련 인사이트 UI

통계와 비교는 목적이 다르므로 페이지를 나눈다.

- `/analytics`: 운동 통계
- `/analytics/compare`: 운동 비교

통계 화면은 다음 질문에 답해야 한다.

1. 선택한 기간의 총 운동량은 얼마인가?
2. 어떤 부위를 가장 많이 했는가?
3. 어떤 운동의 볼륨이 가장 큰가?
4. 최근 4주 볼륨 흐름은 어떤가?

비교 화면은 다음 질문에 답해야 한다.

1. 오늘 보완하면 좋은 부위는 무엇인가?
2. 이번 주는 지난주 같은 기간보다 늘었는가?
3. 같은 운동의 최근 수행량은 좋아졌는가?
4. 어떤 부위가 이전 기간보다 줄었는가?

추천 카드 규칙:

- 추천 부위를 가장 먼저 보여준다.
- 추천 이유는 한 문장으로 제한한다.
- 지난주 대비, 마지막 기록, 이번 주 비중, 기록 없음 중 최소 1개 근거를 보여준다.
- 직접 기록으로 이어지는 버튼은 하나만 둔다.
- 모바일에서는 기록 버튼을 카드 전체 폭으로 둔다.

비교 표시 규칙:

- `+5%` 이상은 증가로 본다.
- `-5%` 이하는 감소로 본다.
- 그 사이는 유지로 본다.
- 세트 수 변화로 볼륨이 증가한 경우에는 그 영향을 문구로 보여준다.
- 비교값을 보여주지 않은 상태에서 성장이라고 단정하지 않는다.

운동별 수행 변화 카드 규칙:

- 운동명, 상태, 최근 날짜, 이전 날짜, 최고 중량, 총 볼륨, 평균 반복을 간결하게 보여준다.
- 가장 중요한 지표 2-3개를 먼저 읽히게 한다.
- 보조 비교값은 작은 메타 텍스트로 둔다.
- 부위 색상은 카드 전체 배경이 아니라 왼쪽 선이나 막대 같은 작은 강조로만 사용한다.
## 분석 화면 리팩토링 UI 기준

통계 화면과 비교 화면은 같은 필터를 공유하지만 서로 다른 질문에 답한다.

통계 화면:

- 총 볼륨, 운동 수, 세트 수, 가장 많이 한 부위를 먼저 보여준다.
- 부위별 볼륨과 운동 랭킹은 목록형 막대로 제공한다.
- 추천이나 판단 문구는 넣지 않는다.

비교 화면:

- 다음 운동 추천을 가장 위에 둔다.
- 이번 주와 지난주 같은 기간 비교를 한 줄 설명과 숫자 카드로 보여준다.
- 운동별 수행 변화는 운동명, 상태, 총 볼륨, 최고 중량, 평균 반복을 카드로 보여준다.

공통 UI:

- 화면 제목과 설명은 한글로 작성한다.
- 버튼과 탭 문구는 짧게 쓴다.
- 카드 안에 들어가는 숫자에는 단위를 붙인다.
- 빈 상태는 사용자가 다음에 할 행동을 알 수 있게 작성한다.
